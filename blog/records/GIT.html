命令行

1. 配置相关

命令 作用
git config --global user.name "Your Name" 设置全局用户名
git config --global user.email "email@example.com" 设置全局邮箱
git config --list 查看当前配置信息

2. 仓库操作

命令 作用
git init 初始化本地仓库
git clone <远程仓库地址> 克隆远程仓库到本地

3. 提交与修改

命令 作用
git add <文件名> 将文件添加到暂存区
git add . 添加所有修改到暂存区
git commit -m "提交说明" 提交暂存区的修改
git commit --amend 修改最后一次提交的说明或内容

4. 分支管理

命令 作用
git branch 查看本地分支列表
git branch <分支名> 创建新分支
git checkout <分支名> 切换到指定分支
git checkout -b <新分支名> 创建并切换到新分支
git merge <分支名> 将指定分支合并到当前分支
git branch -d <分支名> 安全删除本地分支（已合并）
git branch -D <分支名> 强制删除本地分支（未合并）

5. 远程仓库

命令 作用
git remote add origin <远程仓库地址> 关联远程仓库（命名为 origin）
git push -u origin <分支名> 推送本地分支并设置上游跟踪
git push 推送本地提交到远程仓库（已关联上游）
git pull 拉取远程分支并合并到本地（= git fetch + git merge）
git fetch 仅拉取远程更新，不自动合并

6. 撤销与回退

命令 作用
git restore <文件名> 丢弃工作区的修改（Git 2.23+）
git restore --staged <文件名> 将文件移出暂存区（保留工作区修改）
git reset --hard HEAD^ 彻底回退到上一个提交（慎用！会丢失修改）
git reset --soft HEAD^ 回退提交但保留修改到暂存区

7. 查看信息

命令 作用
git status 查看工作区/暂存区状态
git log 查看提交历史
git log --oneline --graph 简洁可视化提交历史
git diff 查看工作区与暂存区的差异
git diff --staged 查看暂存区与最新提交的差异
